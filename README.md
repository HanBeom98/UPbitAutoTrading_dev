# 🚀 업비트 자동매매 시스템

## 1. 프로젝트 소개
본 프로젝트는 **업비트 실시간 시세 데이터**를 기반으로  
매수·매도·부분 익절·손절·트레일링 스탑 등 **전략 기반 매매 자동화 시스템**을 구축하는 것을 목표로 합니다.  
개인 투자 자동화를 위해 개발한 프로젝트이며, 실제 자산으로 운영하며 안정성 및 전략 검증을 경험했습니다.

---

## 2. 프로젝트 배경 및 문제의식
일반 개인 투자자가 주로 겪는 문제는 다음과 같습니다:

- 가격 변동을 실시간으로 지속 관찰하기 어려움  
- 감정에 따른 충동 매매  
- 급격한 시장 변동에 즉각 대응 불가  
- 체계적인 데이터 기록 및 분석 부재  

**→ 이 프로젝트는 이러한 문제를 해결하고,  
일관적·자동화된 매매를 가능하게 하는 시스템을 구축하기 위해 시작되었습니다.**

---

## 3. 프로젝트 목표
### 🎯 핵심 목표
- 실시간 시세 기반 자동 매매
- 명확한 매매 전략의 자동 실행
- 리스크 관리 자동화 (손절·트레일링 스탑)
- 부분 익절을 통한 상승장 수익 최적화
- 주문 체결 추적 및 평균 매수가 계산 정확도 확보
- 매매 로그·성과 데이터 축적 및 리포트 자동화

### ⭐ 프로젝트 성공 지표
- 매매 전략이 의도대로 자동 실행되는가?  
- 지정가 주문 체결 대기 로직이 안정적으로 동작하는가?  
- 부분 익절 및 손절 조건이 실제 상황에서 신뢰성 있게 수행되는가?  
- 로그·데이터 분석이 가능한가?  

---

## 4. 프로젝트 범위 (Scope)

### ✔ 포함되는 범위
- 업비트 WebSocket 실시간 가격 수신
- 매매 전략 엔진 (매수, 매도, 부분 익절, 손절, 트레일링 스탑)
- 지정가/시장가 주문 시스템
- 체결 확인 및 평균 매수가 계산
- 포지션 & 잔고 관리
- 로그/수익률 데이터 저장
- 분석 리포트 자동 생성 (이메일 발송 포함)

### ❌ 제외되는 범위
- UI/프론트엔드 화면
- 백테스팅 시스템 (추후 계획)
- 멀티코인·멀티전략 시스템 (추후 확장)

---

## 5. 핵심 기능 요약

### 🔹 실시간 가격 수신
- 업비트 WebSocket 기반  
- 초당 수천 개 변동에도 안정적 처리  

### 🔹 주문 실행 & 체결 대기
- 시장가/지정가 주문 지원  
- `wait_for_limit_order()`로 지정가 체결 감시  
- 주문 UUID 기반 체결 평균가 계산  

### 🔹 매매 전략 자동화
- 지정 조건 충족 시 자동 매매  
- 주요 전략:
  - 매수 (Buy)
  - 매도 (Sell)
  - 부분 익절 (Sell Partial)
  - 손절 (Stop Loss)
  - 트레일링 스탑 (Trailing Stop)

### 🔹 리스크 관리
- 과매매 방지 쿨다운 시스템
- 단기 급락 대응 손절
- 추세 반전 시 자동 익절

### 🔹 성과 기록 및 리포트
- CSV → DB 방식 전환 예정
- 거래 결과 자동 축적  
- 수익률 그래프 생성 & 메일 발송 기능 포함

## 7. 기대 효과
- 감정 개입 최소화 → 일관성 있는 매매  
- 자동화된 전략 실행으로 대응 속도 향상  
- 실시간 체결 관리로 정확한 평균매수가 유지  
- 구조화된 데이터 기반 분석 가능  
- 수동 투자 대비 효율 증가 및 리스크 감소  

---

## 8. 결론
이 프로젝트는 단순한 자동매매 프로그램이 아니라,  
실제 시장에서 동작한 경험을 기반으로 한 **완성도 높은 자동매매 시스템 구축 사례**입니다.  

프로젝트 발제를 통해 개발 목적·필요성·구조를 명확히 하고,  
이후 PRD/아키텍처/테스트 전략 문서로 이어질 기반을 마련합니다.
